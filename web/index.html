The Skeptical Evaluator

Trust,
but perturb.

One score is a guess. A distribution is evidence.

A reward model that refuses to certify an improvement until it survives repeated, meaning-preserving re-checks — because high variance is the signature of a reward hack.

02 · Problem

A self-improving agent will learn
to fool its own grader.

Point a code-fixing agent at a benchmark and let it iterate on its own score. Under a naive yes/no check, the cheapest way to flip fail→pass isn't to fix the bug — it's to hard-code the one configuration being checked.

The agent isn't malicious, and the hack isn't programmed in. It emerges from what the evaluator rewards. The loop then freezes on it, reporting perfect confidence.

The naive loop believes it resolved 100% of tasks.
It actually resolved 35%.

03 · Solution

Don't measure once.
Measure under perturbation.

Re-run the same candidate fix under several meaning-preserving rewrites of its input — reshuffled issue text, renamed example values, reordered tests. A correct fix doesn't notice. A hack falls apart. So report mean and variance, and certify only when the mean clears a rising bar and the variance is low.

0%100%
μ 0.51σ 0.42
✕ Rejected

Both fixers can post the same single score. Only the spread separates them — which is exactly the information a one-shot evaluator throws away.

A perturbation is valid iff it never touches a file or line the gold patch or test patch edits — so meaning-preservation is checked statically, with no test execution. Two gates ship: a fixed-budget sweep and an adaptive gate that probes one axis at a time in priority order and early-rejects on a confirmed collapse.

06 · Next steps

From modeled to ground truth.

01 · Built, not yet run live

Real grading

The Daytona path is written and import-checked: a sandbox booted from the official SWE-bench instance image, patch applied, official eval script run, graded by SWE-bench's own grader. Next: a live paid round.

02 · Frontier axis

Execution

Reseed, isolate, vary the environment. Catches flakiness and state-leak exploits — the class that survives every input-side perturbation because it keys on the run, not the prompt.

03 · Frontier axis

Specification

Held-out property tests. The residual spec-gaming case that perturbation alone can't reach: a fix that satisfies every stated test and still misses the intent.

04 · Frontier axis

Evaluator

Perturb the judge itself. If an LLM grader can be swayed by phrasing, its score carries the same variance signature — and the same gate applies.

05 · Scale

Full Lite sweep

Run the adaptive gate across all 100 SWE-bench Lite instances with real grading, and report the certified-vs-believed gap on ground truth rather than a model.

06 · Generalize

Beyond code

The gate is domain-agnostic: anywhere a reward model scores an artifact, perturbing the input and reading the variance is a cheap, oracle-free hack detector.
