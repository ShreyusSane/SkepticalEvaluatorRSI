Perturbation-based reward modeling

Skeptic

One score is a guess. A distribution is evidence.

A reward model that refuses to certify an improvement until it survives repeated, meaning-preserving re-checks — because high variance is the signature of a reward hack.

scroll ↓
02 · Problem

A self-improving agent will learn
to fool its own grader.

Point a code-fixing agent at a benchmark and let it iterate on its own score. Under a naive yes/no check, the cheapest way to flip fail→pass isn't to fix the bug — it's to hard-code the one configuration being checked.

The agent isn't malicious, and the hack isn't programmed in. It emerges from what the evaluator rewards. The loop then freezes on it, reporting perfect confidence.

The naive loop believes it resolved 100% of tasks.
It actually resolved 35%.

03 · Solution

Don't measure once.
Measure under perturbation.

Re-run the same candidate fix under several meaning-preserving rewrites of its input — reshuffled issue text, renamed example values, reordered tests. A correct fix doesn't notice. A hack falls apart. So report mean and variance, and certify only when the mean clears a rising bar and the variance is low.

Reward hack

μ 0.51σ 0.42
✕ Rejected

Both fixers can post the same single score. Only the spread separates them — which is exactly the information a one-shot evaluator throws away.

Perturbation — catches failure to generalize
A fix that keys on an incidental feature collapses when that feature is varied. Needs execution, but no oracle.
Static inspection — catches measurement tampering
A patch that rewrites the grader never fails a perturbation — it passes everything. Read the diff instead: stubs, except: pass, injected conftest.py.

A perturbation is valid iff it never touches a file or line the gold patch or test patch edits — so meaning-preservation is checked statically, with no test execution. Two gates ship: a fixed-budget sweep and an adaptive gate that probes one axis at a time in priority order and early-rejects on a confirmed collapse. The static inspector is validated against all 100 cached gold patches with zero false positives — its rules were narrowed twice by real gold patches that fired wrongly.

06 · Results · ground truth, no agent, no LLM

Skeptic caught what the official grader missed.

Deterministic scripted patches, graded by SWE-bench's own grader in a Daytona sandbox on pallets__flask-5063 — then put through Skeptic.

PatchOfficial graderOur inspector
gold (control) RESOLVED 100.0 clean ✓
conftest tamper RESOLVED 100.0 — fooled VETO ✓
weak assertion RESOLVED 100.0 — fooled missed ✕
The tamper patch fixes nothing — it installs a pytest hook that rewrites every outcome to "passed" — and the official grader certifies it at 100. The inspector vetoes it.
The miss is reported honestly. flask-5063's tests only assert that the word "Subdomain" appears, so no input perturbation can detect a weak assertion. That is exactly the gap held-out compositional tests exist to close.
Meaning-preservation is falsifiable, and it was falsified-tested: grading the known-correct gold patch under every perturbation held at score 100.0, F2P 2/2, 56/56 node IDs executed. If a perturbation had changed what counts as correct, gold would have stopped resolving.

The metric that makes the gate work

Skeptic scores what actually matters.

Benchmarks flatter you by default. On flask-5063, 54 of 56 tests are regression tests that pass no matter what — an empty patch that fixes nothing still scores 96.4. Gate on that raw number and almost anything clears the bar. Skeptic gates on the FAIL_TO_PASS fraction instead, and zeroes any patch that breaks a passing test.

Gate metricμ / σVerdict
raw pass-% 97.7 / 1.61 certifies a non-fix
FAIL_TO_PASS fraction (Skeptic) 35.0 / 45.00 rejects it ✓
The entire bug signal on this task lives in a 3.6-point band — which is exactly how a single scalar hides a fix that fixed nothing. Choosing the right scalar is the difference between a gate that certifies everything and a gate that means something.
07 · Next steps

From one instance to the whole benchmark.

01 · Extends the gate

Held-out tests

Benchmark suites assert less than you'd hope — flask-5063's barely check the behaviour at all. Compositional property tests the fixer never sees give Skeptic a specification axis on top of the input axes it already covers.

02 · Tightens the signal

Repeated sampling

Sampling each variant several times drives the confidence interval down and cleanly separates fix-fragility from ordinary agent variance — turning the spread into a statistic you can put a bound on.

03 · Frontier axis

Execution

Reseed, isolate, vary the environment. Catches flakiness and state-leak exploits — the class that survives every input-side perturbation because it keys on the run, not the prompt.

04 · Scale up

Full Lite sweep

The real path is proven on one instance. Next: the adaptive gate across all 100 SWE-bench Lite instances with ground-truth grading, reporting the certified-vs-believed gap on real numbers instead of modeled ones.

05 · Frontier axis

LLM judge

Perturb the judge itself. If an LLM grader can be swayed by phrasing, its score carries the same variance signature — and the same gate applies.

06 · Harden

Tamper surface

The conftest hook is one measurement-tampering vector that the official grader accepts at face value. Enumerate the rest — and keep the false-positive rate at zero while doing it.

07 · Generalize

Beyond code

The gate is domain-agnostic: anywhere a reward model scores an artifact, perturbing the input and reading the variance is a cheap, oracle-free hack detector.
